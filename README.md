### 静态前端连接账号后端

纯 GitHub Pages 没有共享数据库，不能伪装成跨用户作者认证。若前端仍部署在 Pages、账号服务部署在另一个 HTTPS 域名，请把 `index.html`、`profile.html`、`studio.html`、`module.html` 和 `run.html` 中的 `ng-api-base` 填成后端地址（也可在加载 `site-auth.js` 前设置 `window.NG_API_BASE`），并在后端设置允许的精确前端来源：

```powershell
$env:NG_ALLOWED_ORIGINS="https://3250292496-max.github.io"
$env:NG_ACCESS_KEY="你的作品密钥"
npm run start
```

后端会只对 `NG_ALLOWED_ORIGINS` 中的精确来源开放带凭据 CORS，并用 `HttpOnly; Secure; SameSite=None` 会话 Cookie。未配置远端 API 时，页面会探测同源 `/api/auth/me`；确认没有账号服务后才降级为本机静态账号。远端 API 已显式配置但不可用时不会悄悄创建一套本机账号。

## 角色与判定流程

1. 玩家在玩家面板创建或导入角色卡，保存后提交给主持人。
2. 主持人在“战团／角色与判定”中接受或退回角色，也可以导入、导出角色资料。
3. 玩家填写行动目标、做法、专长和风险，发送判定请求。
4. 主持人确定难度、优势／劣势、协助与修正后掷骰，并补充公开后果、代价承担者和结果说明。
5. 玩家面板收到完整公式、最终点数、成功层级和主持人说明；历史记录会保留在浏览器中。

当前实时通讯使用同源 `BroadcastChannel`，适合在同一台设备、同一浏览器来源的已打开标签页之间联调或投屏。跨电脑、跨手机的正式房间同步需要后续接入服务器房间与 WebSocket；在此之前可使用屏幕共享或导入／导出 JSON。

COC7 最短开团流程：玩家在 `coc7.html` 选择内置职业或经守秘人同意自定义职业，再生成／导入角色卡；守秘人在独立的 `combat.html` 战斗台导入角色，勾选参战者创建场景，此后通过攻击、防守、反击与快速结算控件更新角色状态。配套 Excel 文件位于 `assets/rules/COC七版规则空白卡.xlsx`；原文件实际含示例人物“雪莱”，站内因此明确标为“示例模板卡”，使用前请覆盖示例字段。

## 内容与访问保护

公开资料可以直接下载；主模组、NPC 手册、主持人工具书、玩家手卡和分阶段线索等剧透内容需要站长账号或作品密钥。`gm-data.js` 与剧透 DOCX 同样由服务端校验保护。

如果发布到纯静态托管，请生成 AES-256-GCM 加密包并只上传白名单内容：

```powershell
$env:NG_ARCHIVE_KEY="你的长密钥"
npm run build:secure
npm run verify:secure
npm run build:public
npm run verify:public
```

`NG_ARCHIVE_KEY` 优先；未设置时，构建与验证脚本会自动读取不会提交的本地 `.keeper-key`。因此日常更新可以直接运行上述四条 npm 命令，但仍不得把 `.keeper-key` 复制到发布包或仓库。

输出位于 `.secure-publish/`。不要把项目根目录直接上传到公开托管平台，也不要公开 `.data/`、`.keeper-key`、`gm-data.js` 或任何剧透明文资料。

GitHub Pages 会自动使用 `modules-data.js` 展示公开模组和资源；创作者工作台在纯静态环境下使用“浏览器草稿模式”。基本信息、新模组与发布预览可保存在当前浏览器，但附件要让其他访客下载，仍必须通过仓库发布或部署带 `server.mjs` 的后端。静态站长可在个人中心输入账号密码解锁本机保险库，再输入一次现有《零之圣杯》密钥完成加密校验；验证成功后密钥以 AES-GCM 密文保存在当前浏览器，不会写入公开文件。更换设备、清除网站数据或移除本机副本后仍需重新输入现有密钥。

COC7 守秘人规则书 PDF 不会复制进项目或公开发布包。规则页的“选择本地 PDF”只会创建当前浏览器标签可用的本地对象地址，不会上传文件；关闭或刷新页面后需要重新选择。公开包只包含本站整理的规则速查数据与用户提供的示例角色卡（文件名虽为“空白卡”，实际含“雪莱”示例）。

静态站账号只在当前浏览器配置中长期存在，不会跨设备同步；无痕模式、清除网站数据或更换浏览器后需要重新建立账号。静态站不提供虚假的作者申请／审核成功状态；要让其他用户真正认证作者必须连接上述共享后端。公开网页不保存明文密码，也不会用账号登录替代独立守秘口令。
